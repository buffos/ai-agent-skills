# Overview

Interrogate vague or incomplete software product ideas until they are concrete enough to support high-quality planning artifacts. Use when a user wants to build an application but does not yet have a stable PRD or reference design documents and Codex must lead broad, phased discovery by asking targeted questions, surfacing assumptions, pressure-testing workflows, and producing organized discovery notes. Do not use for narrow follow-up on an already mature artifact set; use `requirements-gap-analyzer` instead.
