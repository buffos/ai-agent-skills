# Reference Docs to Issues

Breaks the reference document set into dependency-aware vertical slices.

Use it after the architecture artifacts are stable enough for implementation.
It creates `issues/pending/*.md` plus `issues/issues.md`, grounding each issue
in the PRD, canonical models, contract, scenarios, and readiness findings.
Each issue and registry row carries explicit routing metadata for the owning
capability node, artifact root, and issue file path.
